# Dify-PreProcessor

**Dify-PreProcessor** 是一个高度智能化、专为应对“长文本喂给本地模型 OOM”痛点而生的数据前置增强清洗渠道。

它可以将你的本地 PDF 或 Word 巨型长卷文档，通过精妙的 **全局目录锚定** 和 **跨页滑动窗口 (Map-Reduce)** 技术，进行逐段理解、提炼，最终重组为带有强大层级上下文、不易丢失原始语义的纯粹结构化 Excel 表单。一键无缝接入并增强 Dify 等知识库应用！

---

## 🌟 核心特性概览

* **动态前置目录探测**：突破性引入全局物理目录抓取技术，动态步进探测超长目录。识别到自带目录后，所有后续切片均强制对齐该“全局真理树”，彻底杜绝长文本常见的大纲层级幻觉与畸变。
* **重叠式极光滑动窗口**：全文拆散处理，完美避开了本地超大模型一波流塞入带来的 OOM 惨案。分块时自带 **1 页安全重叠区**，保证上下文绝对连贯。
* **跨页防断层检索**：第二步深度增强时，系统会自动提取底层知识点挂载的**【上一页、当前页、下一页】**完整三页区块给大模型，根除因生硬切页导致的语义断层幻觉。
* **AST 高级去重**：自动清洗并过滤由于滑动窗口重叠切片可能产生的重复叶子节点，极限节省 API Token 和算力。
* **高鲁棒性容灾机制**：任何一次流式拦截、模型由于幻觉没吐出所需格式，系统会自动防抖并进行 `MAX_RETRIES` 原地修正重拉。对于目录超限等极端情况，也会优雅降级到自主盲猜模式。
* **脱离 LLM 的极速报表引擎**：彻底抛弃脆弱且昂贵的大模型 CSV 转换，第四步全权交由 Pandas 在底层毫秒级构建。自适应延展提取出的动态中文表头（如 `标题一`、`标题二`、`关键词`、`摘要`、`内容报告`），强迫症狂喜。

## 🚀 最新架构工作流 (The Pipeline)

程序底层严格执行五大精密控制工序：
0. **工序零（原生/大模型双重探底）**：系统首先启动极速代码级探针，尝试运用 `PyMuPDF` 和 `python-docx` 瞬间剥离物理特征目录。若物理特征被抹平，才会启动大模型，用最新的内容和页码双重兜底判断法在最初 10-50 页动态抓包，提取为全书维度的 Markdown 标准真理树。
1. **工序一（提炼与映射）**：以 5 页为窗口、1 页为重叠步长滑动请求 LLM。要求其像“填空”一样，将文本映射到全局真理树中，强制对齐层级 `#` 号，并输出最长 200 字的第一阶段精简摘要。
2. **工序二（跨页增强）**：向下挖掘所有去重后的“树叶（最底层目录）”，叠加上它的全体父系标题背景，同时喂入该锚点前后共 3 页的物理原文，要求 LLM 一对一产出没有孤立歧义的“重构报告”。
3. **工序三（质检）**：启动质检 LLM 专员，查杀环境幻觉，压榨提取有利于 Dify 倒排/向量检索的 5-10 个精选 `核心标签 (Keywords)`。
4. **工序四（极速落卷）**：绕开耗时的模型构建，直接驱动底座 Pandas，将清洗好的规范节点秒级渲染为拥有 `标题一`、`标题二` 等智能表头的最终版 `xlsx`。

## ⚙️ 快速上手

1. **环境准备**：
   确保你安装了 Python 3.9+，建议在虚拟环境中使用：
   ```bash
   # 新增极致性能可选依赖 PyMuPDF：用于激活 PDF 原生极速书签解析
   pip install -r requirements.txt
   pip install PyMuPDF python-docx
   ```

2. **核心配置 (`config.py`)**：
   在根目录下修改属于你自己的模型配置：
   ```python
   # Ollama 或其他 OpenAI 兼容的 API 入口
   OPENAI_API_BASE = "http://127.0.0.1:11434/v1"
   OPENAI_API_KEY = "ollama"

   # 模型选用
   LLM_MODEL = "qwen3.5:35b-a3b"
   
   MAX_RETRIES = 3 # 容灾重试次数
   ```

3. **放置数据跑批**：
   打开 `main.py`，将 `docs_path` 变量指向你需要解构的超级大部头 PDF 或是长长的法务 `DOCX` 文书。
   ```bash
   python main.py
   ```

4. **验收资产**：
   - 最终成果库：`dify_processed_output/Dify_知识库增强底座.xlsx`
   - 调试与追踪快照库：`dify_intermediate/` （包含原生 TOC、大纲树以及一切中间态断点抓包）
